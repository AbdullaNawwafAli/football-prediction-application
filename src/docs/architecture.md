# Project Architecture

This document describes the folder structure and conventions for this project.
Each major directory also contains its own `README.md` with more detailed guidance.

---

## Stack

- **Frontend** вЂ” React + Vite
- **Routing** вЂ” TanStack Router (file-based, type-safe)
- **Server state** вЂ” TanStack Query
- **Client state** вЂ” Zustand
- **Auth** вЂ” Supabase Auth (Microsoft OAuth)
- **Backend** вЂ” Supabase (Postgres + Edge Functions + Realtime + Storage)
- **Styling** вЂ” shadcn/ui
- **Drag and drop** вЂ” dnd-kit (Feature 1 group stage predictions)
- **Package manager** вЂ” pnpm
- **Type safety** вЂ” Supabase CLI (`supabase gen types`)
- **Serving** вЂ” nginx container (PWA)

## Key Principles

1. **Features are self-contained.** Each feature mirrors the top-level folder structure internally. Start everything inside the feature.
2. **Promote up when shared.** Something moves to a top-level shared folder only when two or more features need it.
3. **No cross-feature imports.** Features never import from each other. Shared code lives at the top level.
4. **`routes/` is thin.** Route files import from `features/` and render вЂ” no business logic in route files.
5. **Export via `index.ts`.** Each feature exposes a public API through its `index.ts`. No deep path imports.
6. **Zustand for client state, TanStack Query for server state.** Do not put server data in Zustand stores. Zustand holds UI state, auth session, and realtime signals. TanStack Query owns all Supabase data fetching and caching.
7. **Supabase Realtime feeds TanStack Query.** On a realtime event, invalidate the relevant TanStack Query cache key вЂ” do not store the realtime payload directly in Zustand.

## Feature Map

| Feature folder         | What it owns                                                    |
| ---------------------- | --------------------------------------------------------------- |
| `auth`                 | Microsoft OAuth flow, session management, route guards          |
| `onboarding`           | Display name uniqueness check, team selection, avatar upload    |
| `dashboard`            | Today's matches widget, top-3 leaderboard snapshots (F1/F2/F3)  |
| `group-predictions`    | Drag-to-order group stage UI (dnd-kit), lock timing, submission |
| `knockout-predictions` | Bracket UI, per-match winner selection, lock timing             |
| `score-predictions`    | Match score input sheet/modal, per-match lock countdown         |
| `leaderboard`          | F1, F2, and F3 leaderboard views, rank calculation display      |

## State Ownership

| Data                          | Owner                                 | Why                                       |
| ----------------------------- | ------------------------------------- | ----------------------------------------- |
| Auth session / user profile   | `stores/auth.store.ts` (Zustand)      | Needed synchronously across the whole app |
| Match list, scores            | TanStack Query                        | Server data, cache + refetch              |
| User predictions              | TanStack Query                        | Server data, tied to auth user            |
| Leaderboard rows              | TanStack Query                        | Invalidated on Supabase Realtime signal   |
| Realtime connection status    | `stores/realtime.store.ts` (Zustand)  | UI-only вЂ” online/offline banner           |
| Drag-and-drop order (unsaved) | Local component state (`useState`)    | Ephemeral until submit                    |

## Supabase Edge Functions

Both functions live in `supabase/functions/` and are deployed via the Supabase CLI. They are standalone Deno modules вЂ” no imports from `src/`.

| Function       | Trigger                                                           | Responsibility                                                                           |
| -------------- | ----------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| `sync-matches` | `pg_cron` (every 60s during live matches, every 10 min otherwise) | Calls football-data.org, upserts into `matches` table                                    |
| `award-points` | Postgres webhook on `matches` row update                          | Scores all predictions for that match, updates `user_points`, broadcasts realtime signal |

## Adding Something New вЂ” Quick Guide

| What you're adding                               | Where it goes                                           |
| ------------------------------------------------ | ------------------------------------------------------- |
| A new page or route                              | `src/routes/`                                           |
| A new feature                                    | `src/features/<feature-name>/`                        |
| A component used only in one feature             | `src/features/<name>/components/`                       |
| A component used in 2+ features                  | `src/components/`                                       |
| A hook used only in one feature                  | `src/features/<name>/hooks/`                            |
| A hook used in 2+ features                       | `src/hooks/`                                            |
| A Supabase query for one feature                 | `src/features/<name>/services/`                         |
| A Supabase query used across features            | `src/services/`                                         |
| A type for one feature                           | `src/features/<name>/types/`                            |
| A shared global type                             | `src/types/`                                            |
| A helper for one feature                         | `src/features/<name>/utils/`                            |
| A shared helper                                  | `src/utils/`                                            |
| A Zustand store                                  | `src/stores/`                                           |
| Supabase client / TanStack Query client          | `src/lib/`                                              |
| Router factory                                   | `src/router.tsx` (or `src/lib/router.ts` when extracted) |
| A new edge function                              | `supabase/functions/<name>/`                            |
| A DB schema change                               | `supabase/migrations/` (via `supabase migration new`)   |
